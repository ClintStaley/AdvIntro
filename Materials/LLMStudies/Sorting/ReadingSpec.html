LLM Study: Sorting

For this LLM-reading assignment, cover these items our schedule didn't allow in class.

1. The various strategies for pivot choosing in Quicksort, and how the code I gave you chooses a pivot

2. The behavior of that Quicksort partition in odd cases, e.g. already-sorted data, reverse-sorted data, all-equal data, and how to improve it to make quicksort remain O(nlogn) under such odd but not uncommon cases.

3. The priority queue versions of our three queues, and how they use Comparators to enforce priority order.

4. An overview (no code needed) of the algorithms merge sort and selection sort, which are the two second-most useful O(nlogn) and O(n^2) sorting algorithms, respectively.

5. The concept of an "inversion", and why a sorting algorithm that removes only one inversion per step cannot be better than O(n^2).

For each of these four conversations, the point is to use the LLM as an interactive textbook or tutor, not just an answer-machine. In particular, your conversation must include:

1. Deeper discussions about each concept, not just a one shot question/answer.

2. Extending on and/or improving the code that was supplied, where relevant.

3. Getting and answering a quiz from the LLM on each of these.

4. Catching the LLM in a significant error, not just some minor syntax or terminology quibble. (I'll settle for doing so in 2 of the 4 topics)
